概要
どんな作品
「作る・選ぶ・遊ぶ」を融合した 60FPS 横スクロールシューティング。 プレイヤーはゲーム開始前にお絵描きツールで自機を描き、 スキルツリーで能力を開放してからステージに挑む。 戦闘中はキーボードで8方向移動しつつ、音声認識で「拡散」「追尾」「ブラックホール」と叫ぶことで 弾の挙動がリアルタイムに切り替わる。
なぜ作った？
高校時代に文字認識を用いた学習アプリを制作した経験から、 認識技術を次はゲーム体験そのものに組み込みたいと考えた。 キーボードだけの操作では得られない 「声が世界を変える」という拡張的な操作感を狙い、 シューティングというジャンルとの親和性を検証するため 1ヶ月集中制作で完成を目指した。

① Python 側：音声認識ワンショット ─ Pvoice.py
PyInstaller で Pvoice.exe にビルドして C++ 側から起動する。 phrase_time_limit=6 で最長 6 秒まで待ち、結果を voice_output.txt に書き出すだけの単純責務にすることで、 C++ 側の読み取りロジックを極力シンプルに保った。 
import speech_recognition as sr
import sys

def recognize_once():
    listener = sr.Recognizer()
    try:
        with sr.Microphone() as source:
            print("Listening...")
            voice = listener.listen(source, phrase_time_limit=6)  # 最長6秒まで受付
            voice_text = listener.recognize_google(voice, language="ja-JP")
            print(voice_text)
            with open("voice_output.txt", "w") as file:
                file.write(voice_text)   # C++ 側はこのファイルを読むだけ
    except sr.UnknownValueError:
        print("すみません、音声を理解できませんでした")
    except sr.RequestError:
        print("音声認識サービスにリクエストを送信できませんでした")

if __name__ == "__main__":
    recognize_once()
② C++ 側：Python 非同期呼び出し ─ shootingGame.cpp / runPythonScriptAsync()
Qキー押下をトリガに Pvoice.exe を別プロセス起動。 std::thread で WaitForSingleObject を待たせることでメインループの 60FPS を一切止めない。 認識結果は std::mutex で保護した recognitionResult に書き戻す。 
void runPythonScriptAsync(void)
{
    isRunning = true;  // 実行中フラグ
    std::thread([]
    {
        STARTUPINFOW si; PROCESS_INFORMATION pi;
        ZeroMemory(&si, sizeof(si));
        si.cb = sizeof(si);
        si.dwFlags = STARTF_USESHOWWINDOW;
        si.wShowWindow = SW_HIDE;   // コマンドプロンプトを非表示

        std::wstring command = L"Pvoice.exe";
        if (CreateProcessW(NULL, &command[0], NULL, NULL, FALSE, 0, NULL, NULL, &si, &pi))
        {
            WaitForSingleObject(pi.hProcess, INFINITE);  // 別スレッドなので安全
            CloseHandle(pi.hProcess);
            CloseHandle(pi.hThread);
        }
        // 認識結果をファイル経由で受け取る
        std::ifstream file("voice_output.txt");
        std::string result;
        if (file.is_open()) { std::getline(file, result); file.close(); }
        else                   { result = "エラー: 結果ファイルが見つかりません"; }
        {
            std::lock_guard<std::mutex> lock(resultMutex);  // スレッド間共有を保護
            recognitionResult = result;
        }
        isRunning = false;
    }).detach();   // デタッチしてメインループへ即復帰
}
③ 認識結果の正規化 ＋ モード切替 ─ normalizeModeName() / SpeechRecognize()
Google Speech API は「連射→電車」のように同音異義で返すことがあるため、 表記ゆれを吸収する正規化関数を挟む。 正規化後の名前で ModeMap 配列を引き、弾・エフェクト・UI ラベルを一括で切り替える。 
// 認識語やスキル名を正規化（誤認識の救済）
std::string normalizeModeName(const std::string& word) {
    std::string w = trim(word);
    if (w == "連射" || w == "連写" || w == "れんしゃ" || w == "電車") return "連射";
    if (w == "追尾" || w == "ついび")                            return "追尾";
    if (w == "拡散" || w == "かくさん")                          return "拡散";
    if (w == "ブラックホール" || w == "ぶらっくほーる")      return "ブラックホール";
    return w;
}
