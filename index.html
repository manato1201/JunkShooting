// 01 · Overview

プロジェクト概要

プレイヤーの音声入力をリアルタイムで認識し、その特性（周波数、音量、音色）をゲーム操作にマッピング。 高周波で敵を狙い、音量で発射などのユニークな音声ゲームメカニクスを実装した実験的シューター。

// 03 · Technical Architecture

技術実装

非同期音声認識 — std::thread::detach() + ファイルブリッジ
C++ メインループ（60FPS）を止めないため、Python Speech Recognition を `std::thread::detach()` で分離実行。 認識結果は `result.json` ファイル経由で `std::mutex` 保護されたポーリングで回収。 レイテンシ 1〜3秒は「敵の次ターンまでの緊張感」としてゲームデザインに組み込み。
FloodFill + PNG書き出し — 自機描画ツール
ゲーム内に 80×64 ドットキャンバスを実装。BFS アルゴリズムで塗りつぶし (FloodFill) を実現し、 `stb_image_write.h` で PNG として書き出し。背景との差分でアルファ値を自動生成し、 描いた自機がそのままゲーム内に登場（愛着形成）。
CSV駆動スキルツリー — データ駆動設計
スキル定義（名前、説明、コスト、HP効果）をすべて CSV で外部管理。 ゲームロジックはパーサーで CSV を読み込み、`ApplySkillTreeStatus()` で状態に反映。 ゲームロジックとデータを疎結合に保ち、スキル追加が設計者だけで対応可能（エンジニア不要）。
音声パラメータ → ゲーム操作の直結
認識した単語（「連射」「ブラックホール」など）をスキル ID に変換し、 その場でアルゴリズム（弾の速度、エフェクト）を動的に変更。 未解放スキル発話時は「ペナルティ発動」などの遊び心ある例外処理を組み込み、 『発話 = 操作の気持ちよさ』を最大化。
